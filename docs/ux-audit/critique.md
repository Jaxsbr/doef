# UX Audit Critique — 2026-03-11

> Generated by reading Playwright screenshots from `docs/ux-audit/`. Each finding references the relevant screenshot(s).

---

## Findings

### C1 — No voice row labels on the staff ❌ CRITICAL
**Screenshots:** 01, 02, 07–10, 11–14
**Severity:** High
**Description:** The percussion staff has no per-row labels. The only text is a generic "Perc" clef label on the far left. Users cannot tell which line or space represents hi-hat, snare, kick, toms, etc. — they would need to memorise standard percussion notation conventions (which vary between publishers) or inspect the source code. A drummer new to this app will click randomly hoping they hit the right voice.
**Fix:** Add voice abbreviation labels (HH, OH, RD, CR, HT, MT, SD, FT, KK) vertically on the left side of the staff, one per voice row, aligned to each voice's staff position.
**Status:** resolved — task #6, voice-label class, 9px fill #555, one per voice row

---

### C2 — Open sheet list has no visual container ⚠️ MEDIUM
**Screenshots:** 18
**Severity:** Medium
**Description:** The "Open" sheet list renders as raw inline block — a "Close" button and a list of sheet-name buttons appearing between the header and the hint bar, with no panel, border, shadow, or visual grouping. This looks broken and pushes content down unpredictably.
**Fix:** Wrap the sheet list in a styled dropdown panel (absolute positioned below the Open button, with border, background, and drop shadow) so it overlays the content rather than displacing it.
**Status:** resolved — task #8, absolute-positioned .sheet-dropdown with border/shadow/z-index; ✕ close; empty state text

---

### C3 — Sticking labels are too small to read ⚠️ MEDIUM
**Screenshots:** 08, 09
**Severity:** Medium
**Description:** R/L sticking labels are rendered at 7px font size above the notehead. At this size they are barely legible at normal display resolution and invisible at a glance.
**Fix:** Increase sticking label font size to at minimum 9–10px, and consider making them bold or using a slightly different colour (e.g. #1d4ed8 blue) to distinguish them from notation ink.
**Status:** resolved — task #10, fontSize 10 bold #1d4ed8 blue

---

### C4 — Metronome button has no active visual state ⚠️ MEDIUM
**Screenshots:** 04, 05
**Severity:** Medium
**Description:** The "Metro: On" and "Metro: Off" button states look identical — same white background, same border, same weight. There is no colour or style change to indicate the metronome is active. Users must read the text label to know the state.
**Fix:** Style "Metro: On" with a distinct active appearance (e.g. filled background, or a coloured border/dot indicator) so the active state is immediately obvious at a glance.
**Status:** resolved — task #12, button[aria-label="Metronome on"] blue fill matching Play/Stop

---

### C5 — Playhead highlights column 0 at rest ℹ️ LOW
**Screenshots:** 01, 03, 07, 11, 15, 17
**Severity:** Low
**Description:** The first column is highlighted in blue even when playback is not running (activeCellIndex appears to be 0 at rest rather than -1). This is unexpected — the blue column suggests something is playing or selected when nothing is.
**Fix:** Ensure `activeCellIndex` is -1 (or equivalent sentinel) when playback is stopped, so the highlight only appears during active playback.
**Status:** resolved — task #14, removed spurious onPlayhead(0,0) call from PlaybackEngine.stop()

---

### C6 — Fourth bar hidden at default viewport width ℹ️ LOW
**Screenshots:** 01, 02
**Severity:** Low
**Description:** At 1280px viewport width with 4 bars at 4/4 time (16 cells × 24px = 384px per bar), only 3 bars fit before the SVG clips. Bar 4 is not visible without horizontal scrolling, and there is no scroll affordance.
**Fix:** Either reduce default cell width (CELL_W) at smaller viewports, or default to 2 bars per row instead of 4, so all bars are always visible.
**Status:** resolved — task #16, BARS_PER_LINE reduced 4→2; 2 bars per row always fit at 1280px
